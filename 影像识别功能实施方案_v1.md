> 需求对齐日期：2026-03-23
> 项目架构：Python/FastAPI（模型层）+ Java/Spring WebFlux（中间层）+ Vue 3（前端）

## 一、需求总结

| 编号 | 需求 | 关键约束 |
|------|------|---------|
| R1 | AI 对话中支持上传图片提问（图片+文字同时发送） | 当前对话框为纯文本，需新增图片上传入口 |
| R2 | 检验报告单识别：OCR 提取 → 异常指标标注解读 → 结合患者病情综合分析 | 需结合 all_info 上下文 |
| R3 | 药品包装识别：药名/规格/厂家 → 适应症/用法/副作用 → 药物相互作用提示 | 需结合患者用药史（如 all_info 中有） |
| R4 | 结果以 SSE 流式输出，与当前问诊体验一致 | 复用现有 SSE 推送机制 |

## 二、技术决策记录

### 2.1 多模态模型选择

| 决策项 | 建议 | 理由 |
|--------|------|------|
| 模型 | `qwen-vl-max` | 需求涉及 OCR + 医学解读 + 综合分析，对视觉理解能力要求高；qwen-vl-plus 更适合简单识别场景 |
| 回退方案 | `qwen-vl-plus` | 如果 vl-max 成本过高或响应太慢，简单的药品识别场景可降级到 vl-plus |

> **重要**：请先在 Claude Code 中确认项目是否已引入 Qwen VL 相关依赖。如果没有，需要新增 DashScope 多模态调用逻辑。

### 2.2 图片传输方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Base64 直传** | 实现简单，不依赖额外存储 | 图片体积膨胀约 33%；大图占用大量内存 |
| **先传 OSS 再传 URL** | 图片可复用和回溯；不占服务端内存 | 多一次上传步骤，链路更长 |

**建议选择：Base64 直传**

理由：
- 场景是一次性分析，图片不需要持久存储或复用
- 检验报告和药品照片通常 1-5MB，Base64 后 1.3-6.7MB，可接受范围
- Qwen VL API 原生支持 Base64 输入，无需额外处理
- 如果未来需要保存历史图片（如患者档案），再加 OSS 存储不迟

### 2.3 前端图片限制建议

| 限制项 | 建议值 | 理由 |
|--------|--------|------|
| 单张大小上限 | 10 MB | Qwen VL 的输入限制；报告单拍照通常 2-5MB |
| 单次上传数量 | 最多 3 张 | 避免上下文过长导致推理质量下降 |
| 支持格式 | jpg / png / webp | 覆盖手机拍照和截图的主流格式 |
| 前端压缩 | 宽/高超过 2048px 时等比缩放 | 减少传输体积，Qwen VL 内部也会缩放 |

## 四、Prompt 设计

### 4.1 检验报告单分析 Prompt

```yaml
# config/prompts.yaml 中新增

image_report_system: |
  你是一位三甲医院神经内科主任医师，正在阅读患者的检验报告单照片。

  ## 任务
  请按以下步骤分析这张检验报告单：

  ### 第一步：OCR 识别
  准确识别报告单上的所有文字和数值，以结构化表格形式列出：
  - 检验项目名称
  - 检测结果（含数值和单位）
  - 参考范围
  - 是否异常（用 ↑ 或 ↓ 标注）

  ### 第二步：异常指标解读
  对所有超出参考范围的指标逐一解读：
  - 该指标的临床意义
  - 可能提示的疾病或状态
  - 偏离程度评估（轻度/中度/重度异常）

  ### 第三步：综合分析
  结合患者已知病情信息（如有），给出：
  - 整体健康状况评估
  - 需要关注的重点指标
  - 建议进一步检查的项目（如有必要）

  ## 安全约束
  - 禁止给出确诊结论，使用"提示""可能""建议"等措辞
  - 建议患者携带报告到相关科室就诊
  - 如果图片模糊无法识别，明确告知用户

image_report_user: |
  {patient_context}

  请分析以下检验报告单图片。
  {user_message}
```

#### 3.2 图片校验

```java
private void validateImages(List<String> images) {
    if (images == null) return;
    if (images.size() > 3) throw new BadRequestException("最多上传 3 张图片");
    for (String img : images) {
        byte[] bytes = Base64.getDecoder().decode(
            img.contains(",") ? img.split(",")[1] : img
        );
        if (bytes.length > 10 * 1024 * 1024) {
            throw new BadRequestException("单张图片不得超过 10MB");
        }
    }
}
```

## 八、注意事项

1. **DashScope SDK 版本**：多模态调用要求 >= 1.14.0，实施前确认并升级
2. **API Key 权限**：确认 DashScope API Key 已开通 qwen-vl-max 的调用权限
3. **Base64 前缀**：Qwen VL API 要求带 `data:image/jpeg;base64,` 前缀，前端压缩时需保留
4. **Nginx 限制**：不改 `client_max_body_size` 则图片上传直接返回 413，这是最易遗漏的坑
5. **安全约束**：所有 Prompt 必须包含免责声明，禁止确诊语气、禁止推荐具体剂量
6. **后续可扩展**：支持 CT/MRI 放射影像；图片分析触发 RAG 检索；历史图片存入 OSS 患者档案
